# MoreLogin Browser Manager

Desktop-приложение на Electron 42 для работы с анти-детект браузером MoreLogin через Local API.

## Требования

- **Node.js 18+** (рекомендуется 20 LTS)
- **npm** или **yarn**
- **MoreLogin Desktop** с включённым Local API (по умолчанию `127.0.0.1:40000`)
- **Windows 10/11**, **macOS 11+** или **Linux** (Ubuntu 20.04+)

## Установка

### 1. Клонировать или распаковать проект

```bash
cd morelogin-browser-manager
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Настроить окружение

Скопируйте `.env.example` в `.env` и при необходимости измените параметры:

```bash
cp .env.example .env
```

Содержимое `.env`:

```ini
MORELOGIN_LOCAL_HOST=127.0.0.1
MORELOGIN_LOCAL_PORT=40000
MORELOGIN_PAGE_SIZE=50
MORELOGIN_AUTO_LOAD=true
MORELOGIN_HEADLESS_START=true
MORELOGIN_DEBUG=false

# Настройки для Cloud API (Open API)
API_MODE=auto
MORELOGIN_API_ID=your_api_id
MORELOGIN_API_KEY=your_api_key
MORELOGIN_OPEN_API_URL=https://api.morelogin.com
```

### 4. Запустить MoreLogin

1. Откройте MoreLogin Desktop
2. Перейдите в **Настройки → Local API** и убедитесь, что API включён
3. Порт по умолчанию — `40000`. Если у вас другой — измените `MORELOGIN_LOCAL_PORT`

### 5. Запустить приложение

```bash
npm start
```

## Использование

### Главное окно

Окно приложения разделено на три зоны:

- **Titlebar** (сверху) — переключение между профилями, импорт из MoreLogin, окно управления
- **Toolbar** — адресная строка, навигация (← → ↺), кнопки настроек (⚙️) и DevTools (🐞)
- **Tab bar** — вкладки текущего профиля
- **Контент** — содержимое активной вкладки (или пустое состояние, если нет профилей)

### Профили

#### Создание локального профиля

1. Нажмите **+** в titlebar
2. Введите имя
3. Откроется пустой профиль с одной вкладкой `about:blank`
4. Введите URL в адресной строке и нажмите Enter

#### Импорт из MoreLogin

1. Нажмите **📥** в titlebar
2. Выберите профиль из списка MoreLogin
3. Дождитесь запуска и синхронизации cookies / localStorage
4. Профиль автоматически станет активным

Каждый импортированный профиль имеет индикатор ● (синяя точка) и `envId` в подсказке.

#### Удаление профиля

Правый клик на профиле в titlebar → **1 — Удалить**, или нажмите **×** на профиле.

### Вкладки

| Действие                  | Способ                              |
| ------------------------- | ----------------------------------- |
| Новая вкладка             | **+** в tab bar или `Ctrl+T`        |
| Переключение              | Клик по вкладке                     |
| Закрытие                  | **×** на вкладке или `Ctrl+W`       |
| Изменение порядка         | Drag-and-drop                       |
| Открытие `target="_blank"`| Автоматически как новая вкладка     |

### Suspend / Resume

Профили, которые не активны больше **2 минут**, автоматически приостанавливаются:

- WebContentsView уничтожается
- MoreLogin-профиль закрывается
- URL/title сохраняются
- В titlebar появляется индикатор ⏸

При повторном клике профиль автоматически восстанавливается.

### Панель файлов

| Действие                | Способ                          |
| ----------------------- | ------------------------------- |
| Открыть / скрыть панель | **📂** или `Ctrl+B`             |
| Выбрать корневую папку  | **📂** в шапке панели           |
| Изменить ширину         | Drag правого края панели        |
| Открыть подпапку        | Клик по строке или **▸**        |
| Архивировать папку      | **📦** на строке папки          |
| Перетащить файл/папку   | Drag из панели → в другое окно  |
| Контекстное меню        | Правый клик                     |

**Контекстное меню** (правый клик на файл/папку):

- Открыть в проводнике
- Скопировать путь
- Открыть в браузере (только для файлов)
- Архивировать в `.zip` (только для папок)
- Открыть через систему

Архивация исключает `node_modules`, `.git`, `dist`, `build` и т.д. (настраивается в ⚙️).

### Настройки

Кнопка **⚙️** открывает оверлей настроек поверх активной вкладки. Доступные поля:

- **Хост** — адрес MoreLogin Local API (по умолчанию `127.0.0.1`)
- **Порт** — порт MoreLogin Local API (по умолчанию `40000`)
- **Размер страницы** — сколько профилей запрашивать за раз
- **Автозапуск** — автоматически восстанавливать сессии при старте
- **Запуск в headless** — не показывать окно MoreLogin
- **Отладка** — подробные логи в консоль main-процесса
- **Исключения архивации** — список имён файлов/папок, которые пропускаются при создании `.zip`

Кнопки:

- **💾 Сохранить** — применяет изменения
- **🔌 Проверить соединение** — тестирует подключение к MoreLogin
- **🔄 Сбросить** — возвращает значения из `.env`

## Структура файлов состояния

Все данные хранятся в пользовательской директории Electron:

| Путь                                  | Содержимое                  |
| ------------------------------------- | --------------------------- |
| `<userData>/app-state.json`           | Профили, вкладки, активный  |
| `<userData>/app-settings.json`        | Пользовательские настройки |
| `<userData>/Local Storage/leveldb`    | Сессии Electron            |
| `<userData>/Network/Cookies`          | Cookies (по профилям)       |

`<userData>` — это:

- **Windows:** `%APPDATA%/morelogin-browser-manager`
- **macOS:** `~/Library/Application Support/morelogin-browser-manager`
- **Linux:** `~/.config/morelogin-browser-manager`

## Безопасность

- Включена **contextIsolation**, отключён **nodeIntegration** на всех WebContentsView
- Песочница (sandbox) включена
- Каждый MoreLogin-профиль изолирован в собственной сессии `persist:ml-<envId>`
- Cookies и localStorage копируются через CDP только при запуске профиля, не утекая в основное хранилище
- IPC работает только через `contextBridge` (`preload.js`)

## Известные ограничения

- IndexedDB пока не копируется автоматически (cookies + localStorage достаточно для большинства случаев)
- Drag-and-drop файлов на WebContentsView принимает только содержимое файла, не сам файл как объект
- На Linux могут потребоваться дополнительные зависимости для Electron (см. [документацию](https://www.electronjs.org/docs/latest/tutorial/linux))

### Структура проекта

```
morelogin-browser-manager/
├── main.js              # main process (BrowserWindow, IPC, профили, CDP)
├── preload.js           # bridge: contextBridge.exposeInMainWorld('api', ...)
├── renderer.js          # логика UI: titlebar, tabs, file tree, settings
├── index.html           # разметка + CSS (всё inline)
├── src/
│   └── services/
│       └── profileService.js  # Local + Cloud API MoreLogin, кэш, диагностика
├── package.json
├── .env                 # MORELOGIN_* переменные
└── .env.example         # шаблон (без реальных ключей)
```

### Как добавить ещё что-то

1. **Новый IPC-хендлер** → `main.js` (`ipcMain.handle(...)`) + `preload.js` (`exposeInMainWorld('api', { ... })`) + `renderer.js` (`api.<name>(...)`)
2. **Новая фича UI** → `renderer.js` + `index.html`
3. **Новая логика MoreLogin** → `src/services/profileService.js` (метод класса `ProfileService`)
4. **Новая настройка** → добавить в `settings = { ... }` в `main.js`, обработать в `loadSettings`/`saveSettings`, отобразить в `index.html` секции `.settings-section`
